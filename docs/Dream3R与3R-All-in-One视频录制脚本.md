# 3R All-in-One 平台操作视频脚本

建议时长：7-8 分钟。

本视频以 3R All-in-One 平台为主线，在平台中重点演示 Dream3R 的任务创建、运行状态、结果产物和模型对比。除非需要证明远程任务确实启动，否则全程不切换终端。

## 本次演示使用的数据

### 平台上传与多模型对比数据

使用项目中已经准备好的 ETH3D `delivery_area` 场景，共 12 张图片：

```text
demo_assets/eth3d_delivery_area_demo/images/
```

这组图片来自同一个静态场景，视角连续，纹理和光照相对稳定，适合演示图片上传、多视角重建和结果对比。建议提前用同一组图片完成以下三个任务：

- MASt3R：作为静态多视角重建基线。
- Spann3R：展示空间记忆式重建结果。
- Fast3R：展示多图像快速重建和运行耗时。

录制时选择其中 3-5 张图片创建一个新任务即可，提前准备的正式对比任务使用完整 12 张图片。

### Dream3R 演示数据

Dream3R 使用平台提供的 `synthetic + kitti` 模式。该模式由程序生成标准候选几何和状态信息，可以稳定展示 Dream3R 的融合过程，并输出点图、置信度、候选权重和运行报告。

Dream3R 的正式 `cache` 模式读取 `.pt` 或 `.pth` 多专家 proposal cache。KITTI 的每条缓存数据包含 Fast3R、MASt3R、Spann3R 对同一场景产生的候选；ETH3D 还包含 VGGT-Omega。缓存文件是这些同场景结果的容器，普通任务导出的点云或模型权重不能直接替代。录制时可使用平台内置缓存稳定展示正式融合流程。

### 对比时如何说明

ETH3D 任务用于比较 MASt3R、Spann3R 和 Fast3R 在相同图像输入下的运行情况。Dream3R 展示的是候选结果融合功能，输入形式不同，因此在平台中展示它的任务记录、耗时和输出产物，不把它与图像重建模型做重建质量排名。

## 录制前准备

- 启动 3R All-in-One，确认后端和远程服务器状态正常。
- 准备一个已经完成的 Dream3R 任务。
- 使用完整的 12 张 ETH3D `delivery_area` 图片，提前完成 MASt3R、Spann3R 和 Fast3R 三个任务。
- 确保这三个任务使用相同的 sample ID，例如 `eth3d_delivery_area_demo`，便于样例矩阵自动归组。
- 确认 Dream3R 任务中有运行日志、`dream3r_report.json`、`scene_meta.json` 和融合结果。
- 确认“样例矩阵”或“对比面板”中能找到上述任务。
- 清理失败或内容不完整的测试任务，避免录制时误点。
- 关闭消息通知，隐藏服务器地址、用户名和密码。
- 软件窗口最大化，缩放使用 100% 或 125%。

Dream3R 的功能是对前序模型给出的候选三维结果进行状态条件化融合。它会综合候选几何和对应的状态信息，输出融合后的点图、置信度以及候选权重。

当前版本支持 proposal cache 正式输入，也提供 synthetic 模式，用于快速验证模型环境和整条运行通路。接下来直接在平台中创建一个 Dream3R 任务。

## 第四段：创建 Dream3R 任务

时间：2:00-3:05

### 屏幕操作

1. 点击“新建任务”。
2. 在模型列表中选择 Dream3R。
3. 展示模型说明和输入要求。
4. 运行模式选择 `synthetic`。
5. 场景域选择 `kitti`。
6. 参数模板选择“标准”或保持标准默认值。
7. 展示 seed、batch、views、patches 和 device 等参数，不逐项修改。
8. 在实验说明中输入“Dream3R 平台功能演示”。
9. 点击提交任务。

### 旁白

现在新建一个 Dream3R 任务。

选择模型以后，平台会自动显示与 Dream3R 对应的输入方式和参数。这里选择 synthetic 模式和 KITTI 场景域，参数使用标准模板，并填写本次实验说明。

正式 cache 模式需要选择已经构建好的同场景多专家 proposal cache。平台内置 KITTI 和 ETH3D 标准缓存，也支持导入符合 Dream3R 数据结构的外部缓存包。synthetic 模式用于不依赖外部数据的通路演示。

点击提交后，平台会生成任务记录，并按照 Dream3R 的模型配置将任务派发到远程环境。
