# 3R All-in-One 平台操作视频脚本

建议时长：7-8 分钟。

本视频以 3R All-in-One 平台为主线，在平台中重点演示 Dream3R 的任务创建、运行状态、结果产物和模型对比。除非需要证明远程任务确实启动，否则全程不切换终端。

## 录制前准备

- 启动 3R All-in-One，确认后端和远程服务器状态正常。
- 准备一个已经完成的 Dream3R 任务。
- 准备同一样例下至少两个其他模型的完成任务，用于对比。
- 确认 Dream3R 任务中有运行日志、`dream3r_report.json`、`scene_meta.json` 和融合结果。
- 确认“样例矩阵”或“对比面板”中能找到上述任务。
- 清理失败或内容不完整的测试任务，避免录制时误点。
- 关闭消息通知，隐藏服务器地址、用户名和密码。
- 软件窗口最大化，缩放使用 100% 或 125%。

## 第一段：平台首页与项目定位

时间：0:00-0:40

### 屏幕操作

打开 3R All-in-One，停留在“工作队列”。缓慢移动鼠标，展示任务列表、状态筛选和任务数量。

### 旁白

大家好，下面演示我们开发的 3R All-in-One 三维重建实验管理平台。

平台用于统一管理不同 3R 模型的运行流程。用户可以在这里选择模型、创建任务、调用远程 GPU、查看运行状态和结果，并对不同模型的实验进行整理和比较。

本次视频会以 Dream3R 为重点，完整演示一个模型从创建任务到查看结果，再到多模型对比的过程。

## 第二段：平台功能总览

时间：0:40-1:20

### 屏幕操作

鼠标沿左侧菜单从上到下移动，每个入口停留一秒：

1. 工作队列
2. 新建任务
3. 对比面板
4. 样例矩阵
5. Agent 编排
6. 实验编排
7. 系统配置
8. 仪表板

随后点击“仪表板”或保留在“工作队列”，展示平台整体状态。

### 旁白

平台主要包括工作队列、新建任务、对比面板、样例矩阵、Agent 编排、实验编排和系统配置等模块。

工作队列负责跟踪每一次模型运行；新建任务用于选择模型、输入和参数；对比面板与样例矩阵用于整理不同模型的结果；Agent 编排负责登记模型环境和运行规则；系统配置则用于管理后端、远程服务器和计算资源。

下面先检查 Dream3R 在平台中的接入状态。

## 第三段：Dream3R 模型接入信息

时间：1:20-2:00

### 屏幕操作

1. 点击“Agent 编排”。
2. 在模型列表中找到 Dream3R。
3. 展示模型版本、运行状态、环境、默认参数和健康检查信息。
4. 如果有模型校验按钮，展示已有校验结果，不要在录制时执行耗时构建。

### 旁白

这里是 Agent 编排页面。Dream3R 已经作为独立模型登记到平台中，平台保存了它的版本、运行环境、默认参数、健康检查和输出要求。

Dream3R 的功能是对前序模型给出的候选三维结果进行状态条件化融合。它会综合候选几何和对应的状态信息，输出融合后的点图、置信度以及候选权重。

当前版本支持 proposal cache 正式输入，也提供 synthetic 模式，用于快速验证模型环境和整条运行通路。接下来直接在平台中创建一个 Dream3R 任务。

---

## 第四段：创建 Dream3R 任务

时间：2:00-3:05

### 屏幕操作

1. 点击“新建任务”。
2. 在模型列表中选择 Dream3R。
3. 展示模型说明和输入要求。
4. 运行模式选择 `synthetic`。
5. 场景域选择 `kitti`。
6. 参数模板选择“标准”或保持标准默认值。
7. 展示 seed、batch、views、patches 和 device 等参数，不逐项修改。
8. 在实验说明中输入“Dream3R 平台功能演示”。
9. 点击提交任务。

### 旁白

现在新建一个 Dream3R 任务。

选择模型以后，平台会自动显示与 Dream3R 对应的输入方式和参数。这里选择 synthetic 模式和 KITTI 场景域，参数使用标准模板，并填写本次实验说明。

如果使用正式的 cache 模式，还需要上传前序 3R 模型生成的 proposal cache 文件。本次使用 synthetic 模式，可以在不依赖额外输入文件的情况下快速展示 Dream3R 的融合流程。

点击提交后，平台会生成任务记录，并按照 Dream3R 的模型配置将任务派发到远程环境。

---

## 第五段：工作队列与运行状态

时间：3:05-3:55

### 屏幕操作

1. 提交后自动或手动回到“工作队列”。
2. 找到刚创建的 Dream3R 任务。
3. 指向任务状态、进度和创建时间。
4. 使用搜索框输入 `Dream3R`。
5. 展示状态筛选功能。
6. 随后选择提前准备好的已完成 Dream3R 任务。

### 旁白

任务提交以后会进入工作队列。这里可以查看模型名称、任务状态、运行进度和创建时间，也可以按照关键词和状态快速筛选任务。

平台后端会完成输入准备、远程上传、模型执行、结果回传和状态更新。整个过程会保存在同一条任务记录中。

完整运行需要一定时间，所以这里切换到一个已经完成的 Dream3R 任务，继续查看它的运行结果。

## 第六段：Dream3R 任务详情与结果

时间：3:55-5:10

### 屏幕操作

1. 打开已完成的 Dream3R 任务详情。
2. 展示任务基本信息和运行参数。
3. 切换到日志区域，缓慢滚动几行。
4. 展示结果检查或输出合同状态。
5. 依次展示：
   - `dream3r_report.json`
   - `scene_meta.json`
   - `fused_pointmap.npy`
   - `fused_confidence.npy`
   - `expert_weights.npy`
6. 展示结果下载或“实验记录包”按钮。

### 旁白

进入任务详情后，首先可以看到本次实验使用的模型、参数、运行时间和任务状态。

日志区域保存了 Dream3R 在远程环境中的执行过程。任务出现问题时，可以直接在这里查看失败阶段和错误信息，不需要再到服务器中逐层寻找日志。

结果区域按照 Dream3R 的输出要求整理产物。运行报告记录模型版本、场景域和输出信息；`scene_meta` 保存平台统一的任务元数据；另外三个文件分别是融合后的点图、置信度和候选权重。

平台会检查必要产物是否完整，并可以将输入、参数、日志和结果一起导出为实验记录包，方便后续复查和复现。

---

## 第七段：样例矩阵与多模型对比

时间：5:10-6:25

### 屏幕操作

1. 点击“样例矩阵”。
2. 选择已经准备好的公共样例。
3. 展示 Dream3R 和其他模型对应的任务状态或结果入口。
4. 点击“对比面板”。
5. 勾选或打开 2-3 个已完成模型任务。
6. 展示并排结果预览、指标条形图或雷达图。
7. 如果支持报告导出，指向导出按钮。

### 旁白

完成单个任务以后，可以通过样例矩阵查看同一样例在不同模型下的运行情况。平台目前登记了 Dream3R、DUSt3R、MASt3R、MonST3R、Spann3R、Fast3R、Align3R 和 CUT3R 等模型。

在对比面板中，可以选择多个已完成任务，集中查看结果预览和已有指标。这里展示的是 Dream3R 与其他模型的实验记录。

需要注意，Dream3R 承担的是候选结果融合，其他模型可能直接负责图像重建、匹配或时序处理，因此这里的对比重点包括运行流程、产物类型、处理时间和结果记录，不把不同任务形式的指标简单地混为一谈。

对比结果还可以整理成报告，为后续实验分析和汇报提供材料。

---

## 第八段：实验编排与参数模板

时间：6:25-7:05

### 屏幕操作

1. 点击“实验编排”。
2. 展示批量选择模型、样例或参数组合的区域。
3. 展示已有实验记录，不现场启动大量任务。
4. 如果新建任务页面有参数模板，返回后短暂展示模板保存与复用入口。

### 旁白

当需要运行多个模型或多组参数时，可以使用实验编排功能批量组织任务。每个任务仍然保留独立的输入、参数、状态和结果，完成后再进入样例矩阵和对比面板进行整理。

常用参数还可以保存为模板，减少重复填写，也可以保证多次实验采用一致的配置。

---

## 第九段：系统状态与结束

时间：7:05-7:40

### 屏幕操作

1. 点击“系统配置”，展示后端、远程服务器或资源状态。
2. 回到“工作队列”。
3. 选中已完成的 Dream3R 任务并保持画面稳定。

### 旁白

系统配置页面用于查看本地后端、远程服务器和计算资源状态，为任务派发和问题排查提供依据。

以上就是 3R All-in-One 的主要操作。平台将模型接入、任务创建、远程运行、日志查看、结果管理和多模型对比连接成了完整流程。Dream3R 也通过同一套流程完成运行和结果归档。

本次演示到这里，谢谢观看。

## 录制异常备用话术

### 新任务尚未完成

三维模型需要在远程 GPU 上计算。当前任务已经成功进入队列，下面切换到相同配置的已完成任务查看结果。

### 远程服务器临时断开

当前远程计算节点暂时没有连接，但平台完整保留了历史任务和结果。下面使用已完成任务继续展示结果管理和对比功能。

### 对比页面数据不完整

模型对比需要提前完成同一样例下的多组任务。本次先展示平台的任务组织、结果入口和对比方式。
