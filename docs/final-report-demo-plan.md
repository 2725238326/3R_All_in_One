# 结题报告、PPT 和软件演示精简版

## 1. 结题怎么定位

一句话：

> 3R All-in-One 是一个三维重建实验管理平台，用来统一管理模型、任务、远程运行、结果查看和实验记录。

不要把项目说成“完整提出并验证了新算法”。
更稳的说法是：

> 本项目把多个三维重建模型的实验流程整理成一个软件平台，让模型运行、结果查看和实验记录更加规范。

## 2. 报告怎么写

### 摘要

写 300-500 字，说明：

- 三维重建模型多，手工复现实验麻烦。
- 本项目做了一个统一管理平台。
- 平台支持模型登记、创建任务、远程运行、查看结果、实验记录。
- 系统通过了测试和发布检查。

可直接使用：

> 本项目面向三维重建模型复现和实验管理困难的问题，设计并实现了 3R All-in-One 多模型实验管理平台。系统支持 DUSt3R、MASt3R、MonST3R、Spann3R、Fast3R、Align3R、CUT3R 等模型的统一管理，提供任务创建、远程运行、结果回传、日志查看、结果检查、实验编排和报告导出等功能。平台将原本分散在命令行、远程服务器和结果文件夹中的实验流程整理为可查看、可追踪、可复现的软件流程。测试结果表明，系统通过了模型蓝图校验、后端测试、前端构建和发布检查，具备结题演示和后续扩展基础。

### 第 1 章 绪论

写为什么要做：

- 三维重建模型发展快。
- 不同模型环境、脚本、输入输出差异大。
- 手工运行和整理结果很麻烦。
- 所以需要一个统一的软件平台。

### 第 2 章 需求分析

写要解决哪些问题：

| 问题 | 系统怎么解决 |
| --- | --- |
| 模型多，管理乱 | 建立模型列表和模型状态 |
| 远程运行麻烦 | 通过 SSH/SCP 调用远程 GPU |
| 任务过程难追踪 | 每次运行都保存成任务 |
| 结果文件不统一 | 在结果页面统一查看输出和日志 |
| 实验难复现 | 保存输入、参数、模型和结果记录 |

### 第 3 章 系统设计

写三层结构：

```text
前端界面
  ↓
本地后端
  ↓
远程 GPU 服务器
  ↓
结果回传
  ↓
前端查看和导出
```

解释：

- 前端：创建任务、查看状态、展示结果。
- 后端：保存任务、调用远程服务器、整理结果。
- 远程 GPU：运行真正的三维重建模型。

### 第 4 章 功能实现

按模块写：

1. 工作队列：查看任务、筛选状态、取消或重试任务。
2. 新建任务：选择模型、上传图片或视频、设置参数。
3. 远程运行：上传输入、执行模型、下载结果、保存日志。
4. 模型管理：登记 7 个模型，记录环境、权重和运行状态。
5. 结果审查：查看输出文件、日志、结果检查和下载包。
6. 模型检查模块：检查模型说明、环境、权重和最小测试。
7. 实验编排：批量创建任务，方便多模型和多参数对比。
8. 对比与报告：对比不同模型结果，导出报告或实验记录。

### 第 5 章 测试与验证

直接写验证结果：

```text
Python 测试：167 passed, 1 skipped
前端构建：通过
模型蓝图校验：7/7 valid
release_check：核心检查通过
Docker：本机未安装 Docker CLI，因此只有 warning，不影响主流程
```

### 第 6 章 开发过程

老师说的五个阶段可以这样写：

1. 需求分析阶段：分析多模型三维重建实验管理的痛点。
2. 系统设计阶段：确定前端、本地后端、远程 GPU 三层结构。
3. 功能实现阶段：实现任务管理、远程运行、模型登记、结果审查、模型检查和实验编排。
4. 测试验证阶段：运行后端测试、前端构建、模型校验和发布检查。
5. 部署演示阶段：准备软件界面、测试结果、任务样例和答辩演示流程。

### 第 7 章 总结与展望

总结：

- 完成了多模型三维重建实验管理平台。
- 实现了创建任务、远程运行、结果查看、记录导出的流程。
- 通过测试和发布检查证明系统基本可用。

展望：

- 增加更多真实数据集测试。
- 完善结果展示页面。
- 补充更完整的模型对比结果。
- 支持更多三维重建模型。

## 4. 现场怎么展示

展示顺序：

1. 打开软件首页。
2. 展示工作队列。
3. 点开一个已完成任务。
4. 展示结果文件、日志和检查结果。
5. 展示模型检查页面，说明 7 个模型可以检查。
6. 展示实验编排页面。
7. 展示测试结果截图。
8. 回到 PPT 总结。

现场不要依赖完整模型实时跑完。
提前准备好已完成任务和截图。

## 5. 最重要的准备

- 一个已完成任务。
- 模型校验 7/7 valid 截图。
- release_check 截图。
- 工作队列、结果页、模型检查页、实验编排页截图。
- 10-12 页 PPT。
