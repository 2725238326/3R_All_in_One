## 每页内容

### 1. 封面

标题：

> 3R All-in-One 多模型三维重建实验管理平台

副标题：

> 面向 3R 模型复现、远端运行、结果归档和实验对比的软件工具

配图：软件首页或工作队列截图。封面不要放太多技术词。

### 2. 为什么要做这个软件

标题：

> 3R 模型能跑，不等于实验流程可管理

左侧写“原始方式”：

```text
找模型仓库 -> 配环境 -> 写命令 -> 传文件 -> 看日志 -> 找输出 -> 手工整理结果
```

右侧写“平台方式”：

```text
选模型 -> 上传输入 -> 远端执行 -> 自动回传 -> 结果审查 -> 实验记录
```

讲法：

> 多个三维重建模型的环境、脚本、输入输出都不一样。项目要解决的不是单个模型能不能跑，而是让这些实验过程可以统一创建、追踪、查看和复查。

### 3. 系统怎么设计

标题：

> 本地负责管理，远端负责计算，结果回到本地归档

架构图必须出现：

```text
Tauri + React 桌面端
        ↓
FastAPI 本地后端
  Job Store / Scheduler / SSH Runner / Model Contracts
        ↓ SSH/SCP
远端 GPU Server
  conda envs / model weights / runners / jobs
        ↓
结果回传到本地软件
```

讲法：

> 前端负责交互和结果展示，后端负责任务状态、模型合同和远端调度，远端服务器负责真正的模型推理。这样本地软件不用承载复杂模型环境，但仍然能统一管理任务和结果。

### 4. 软件怎么实际使用

标题：

> 一次实验从创建任务到查看结果都在同一软件内完成

放 4 张小截图：

1. 新建任务页：选择模型和参数。
2. 工作队列页：显示任务状态。
3. 任务详情页：输出、日志、合同检查。
4. 结果/对比/报告页：展示结果复盘能力。

每张截图只标一个重点。不要堆说明文字。

讲法：

> 用户实际使用时，不需要切换命令行和远端文件夹。软件里选择模型、提交任务，远端运行后结果自动回传，最后在任务详情里查看输出和日志。

### 5. 模型怎么接入

标题：

> 新模型接入被拆成蓝图、注册表、runner 和输出合同

流程图：

```text
Agent YAML 蓝图
  -> backend/model_registry.py
  -> runners/<model>_runner.py
  -> backend/model_contracts.py
  -> 前端模型选择 / 任务创建 / 结果检查
```

右侧放模型表：

| 模型 | 平台定位 | 状态 |
| --- | --- | --- |
| DUSt3R | 静态重建基线 | baseline |
| MASt3R | 静态匹配/重建 | validated smoke |
| MonST3R | 动态视频样本 | validated sample |
| Spann3R | 空间记忆路线 | validated smoke |
| Fast3R | 多图快速重建 | attention fallback |
| Align3R | 对齐重建路线 | runner ready |
| CUT3R | 流式状态路线 | validated smoke |
| Dream3R | 候选几何融合 | v1.1 synthetic/cache demo |

讲法：

> 平台不是把每个模型硬写死在页面里，而是把模型接入拆成几个稳定接口。这样后续加模型时，主要补蓝图、runner 和输出合同。

### 6. Dream3R 怎么接入

标题：

> Dream3R 接入的是候选几何融合流程

流程图：

```text
synthetic demo / proposal-cache 文件
        ↓
Dream3R v1.1 runner
        ↓
状态条件化候选几何融合
        ↓
scene_meta.json + dream3r_report.json + 融合产物
        ↓
平台结果审查
```

左侧放 Dream3R 参数截图，能看到：

- `demo_mode = synthetic/cache`
- `domain = KITTI/ETH3D`
- `device = auto/cpu/cuda`

讲法：

> Dream3R 不是普通的图片输入重建模型。它在平台中定位为候选几何融合模型。synthetic 模式用于稳定演示，cache 模式使用上传的 `.pt/.pth` proposal-cache 文件。
